Features Need Stories

Authors

Sidney C. Bailin

Notes

They argue that features themselves are not sufficiently rich representations, but mapping user stories to features allows fine-grained implementation questions to be answered. They report on a case study, which is oriented around the choice of line protocol for dispatching structured information between two systems. The emphasis is on a metric of 'compression' and the fact that there are many dimensions and inconsistent test results which can only be picked apart by appeal to stories.

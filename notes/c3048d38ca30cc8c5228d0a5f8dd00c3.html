A Proposal for an Embedded Systems Pattern Language

Authors

David E. DeLano 

Status

Scanned

Relevance

Proposes a pattern language for embedded systems. Acts as a demonstration how within computer science pattern languages are intended for the consumption of programmers, rather than as Alexander intended - a means of engagement with a collaborative group. Embedded systems, for DeLano does not mean a system which interacts with a user, which is a shame as I'm sure many of them do.

Quotations

"I would like to see a Pattern Language developed that covers the essence of building embedded systems.
This Pattern Language should take into account existing literature, as many of the Patterns already written
should find homes within the Pattern Language. I am especially interested in work on the part of the
Pattern Language that would flesh out OS Abstraction Layer."

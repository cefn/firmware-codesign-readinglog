What is a requirement?

A property that a system must exhibit in order for it to satisfy the user.

  • The property is what is required.
  • How it is exhibited varies according to the type of requirement
  • Users may not be the only agent the system must satisfy (could be another system), and not all users are the same.

Functional vs. Non-functional requirements

A 'pete sawyer' definition of software requirements.

Requirements may conflict, be higher priority than others, (incoming phone call versus camera use) cost more than others.


Use Cases

Role of use cases in 'requirements discovery'.

  • from our understanding
  • don't ask what user's want

Example of camera phone - taking picture when call comes in - work with stakeholders

Use case 'describes how a system responds to stimuli from 'outside' the system

  • as a diagram
  • as a set of descriptions

Use Case diagram contains Actors, Use cases, System boundary (what you're implementing - e.g. database could be outside). Each 'use case' is an ellipse containing a verb and a noun.

Requirements are extracted from use cases, and translate into the system's own terms, with more detail.

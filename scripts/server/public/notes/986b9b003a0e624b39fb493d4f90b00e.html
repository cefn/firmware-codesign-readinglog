Heedless programming: ignoring detectable error is a widespread hazard

Authors

Harold Thimbleby

Notes

Heedless programming is, roughly, the lack of attendance to corner cases and exceptions, given a name so it can be critiqued. That includes proceeding when ambiguous or mistaken input has been provided, or failing to handle states which the system can get itself into. Calculators are provided as a canonical example.
